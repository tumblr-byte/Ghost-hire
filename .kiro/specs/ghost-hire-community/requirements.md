# Ghost Hire Community Features - Requirements

## Introduction

Ghost Hire Community adds two social features for self-taught developers to connect authentically:
1. **The Roast Zone** - Post projects and get real, constructive criticism (not fake positivity)
2. **Ghost Squad** - Chat and form hackathon teams based on skills and vibe (friendship over networking)

## Glossary

- **Roast Zone**: A feed where users post projects/code and receive honest, constructive feedback
- **Ghost Crew**: A group of developers (max 5) working together on hackathons
- **The Graveyard**: Feed where users post projects and get Ghost Chants (feedback)
- **Ghost Chant**: Honest comment on a project - can be praise, constructive criticism, or helpful feedback (Gen Z slang encouraged!)
- **Crew Invitation**: Request to join someone's Ghost Crew
- **Ghost Hunt**: Page to discover other developers and hunt for crew members
- **The Summoning Circle**: Job board where companies summon talent by SKILLS (no degree required!)
- **Summon**: When a company hires a developer
- **Haunt This Job**: Apply to a job posting

## Requirements

### Requirement 1: The Roast Zone

**User Story:** As a self-taught developer, I want to post my projects and get REAL feedback (not fake LinkedIn positivity), so that I can actually improve my skills

#### Acceptance Criteria

1. THE Ghost Hire System SHALL allow verified users to create posts with title, description, code snippets, and project links
2. THE Ghost Hire System SHALL display posts in a feed sorted by recent activity
3. THE Ghost Hire System SHALL allow users to comment on posts with constructive criticism
4. THE Ghost Hire System SHALL use AI to detect and flag fake positive comments (e.g., "great work!", "amazing!")
5. THE Ghost Hire System SHALL encourage roasting with prompts like "What could be better?" and "Be brutally honest"
6. THE Ghost Hire System SHALL display a "Roast Score" showing how helpful the feedback is
7. THE Ghost Hire System SHALL allow post authors to mark comments as "Actually Helpful"

### Requirement 2: Ghost Crew Formation

**User Story:** As a solo developer, I want to browse other developers' profiles and invite them to my Ghost Crew, so that I can build a hackathon team with real skills

#### Acceptance Criteria

1. THE Ghost Hire System SHALL provide a "Crew Browser" page showing all verified developers
2. THE Ghost Hire System SHALL display each developer's skills, GitHub stats, and Devpost wins
3. THE Ghost Hire System SHALL allow users to send "Crew Invitations" to other developers
4. THE Ghost Hire System SHALL allow users to create a Ghost Crew (max 5 members)
5. THE Ghost Hire System SHALL show "Skill Match" score when browsing potential crew members
6. THE Ghost Hire System SHALL allow crew members to accept/decline invitations
7. THE Ghost Hire System SHALL provide crew chat for coordination

### Requirement 3: Ghost Crew Chat

**User Story:** As a Ghost Crew member, I want to chat with my crew in real-time, so that we can plan hackathons and build together

#### Acceptance Criteria

1. THE Ghost Hire System SHALL provide a chat interface for each Ghost Crew
2. THE Ghost Hire System SHALL allow crew members to share code, links, and files
3. THE Ghost Hire System SHALL show online/offline status of crew members
4. THE Ghost Hire System SHALL send notifications for new crew messages
5. THE Ghost Hire System SHALL allow crew members to leave the crew
6. THE Ghost Hire System SHALL prevent corporate/networking spam with AI moderation
7. THE Ghost Hire System SHALL emphasize friendship and learning over professional networking

### Requirement 3: Anti-Corporate Culture

**User Story:** As a Gen Z developer, I want a space that respects me as a human (not a "resource"), so that I can connect authentically without corporate BS

#### Acceptance Criteria

1. THE Ghost Hire System SHALL use casual, fun language (no corporate jargon)
2. THE Ghost Hire System SHALL ban terms like "synergy", "leverage", "circle back", "touch base"
3. THE Ghost Hire System SHALL detect and flag networking spam
4. THE Ghost Hire System SHALL promote genuine connections over transactional relationships
5. THE Ghost Hire System SHALL allow users to set boundaries (e.g., "No recruiters", "Just here to learn")

### Requirement 4: Skill-Based Crew Matching

**User Story:** As a developer looking for teammates, I want to find people with complementary skills, so that we can build better projects together

#### Acceptance Criteria

1. THE Ghost Hire System SHALL analyze user skills from their GitHub/Devpost
2. THE Ghost Hire System SHALL suggest crew members with complementary skills
3. THE Ghost Hire System SHALL show skill overlap and gaps in crew formation
4. THE Ghost Hire System SHALL display "Skill Match %" when browsing profiles
5. THE Ghost Hire System SHALL recommend hackathons based on crew skills
6. THE Ghost Hire System SHALL track crew success (projects completed, hackathons won)

### Requirement 5: Crew Browser & Discovery

**User Story:** As a developer, I want to browse other developers' haunted portfolios and see their real skills, so that I can find the perfect crew members

#### Acceptance Criteria

1. THE Ghost Hire System SHALL provide a "Crew Browser" page with all verified developers
2. THE Ghost Hire System SHALL allow filtering by skills, languages, and hackathon experience
3. THE Ghost Hire System SHALL show each developer's Ghost Power Level and stats
4. THE Ghost Hire System SHALL display "Skill Match %" based on complementary skills
5. THE Ghost Hire System SHALL allow users to send crew invitations from the browser
6. THE Ghost Hire System SHALL show if a developer is already in a crew
7. THE Ghost Hire System SHALL recommend "Best Matches" based on skills and vibe
